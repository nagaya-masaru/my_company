## 2. チャネル別戦略

### 2-1. Meta (Facebook + Instagram) — メイン

**理由**: 中堅企業オーナー (40-55歳・男性多め) のリーチが最大。LinkedInより広告単価が安く、X より滞在時間が長い。

| 項目 | 設定 |
|------|------|
| **目的** | コンバージョン (LP申込) + リターゲ |
| **配置** | Facebook News Feed / Instagram Feed / Instagram Stories |
| **入札** | CPC 自動入札 → 学習完了後 tCPA |
| **予算配分** | 全広告予算の **50%** |
| **クリエイティブ数** | 4種同時 (動画2 + 静止画2) |
| **オーディエンス** | (1) カスタム: CEO実名SNSフォロワー類似 / (2) 興味関心: 経営者・MBA・コンサル・AI系 |

**ABテスト変数 (Month 1)**:
- A: CEO顔出しTalking Head 動画 30秒 vs B: スライド型ベネフィット動画 30秒
- A: 価格(50万)を最初に出す vs B: 課題提起→Reveal型
- A: CTA「無料相談」 vs B: CTA「資料請求」 vs C: CTA「お問い合わせ」

### 2-2. X (旧Twitter) Ads — サブ

**理由**: 経営者・AI業界キーマン・メディア関係者にリーチ。フォロワー類似拡張で「経営/AI/起業」興味層へ。

| 項目 | 設定 |
|------|------|
| **目的** | プロモートツイートで CEO 投稿の拡散 + LP誘導 |
| **配置** | タイムライン + プロフィール広告 |
| **入札** | CPM 入札 |
| **予算配分** | 全広告予算の **20%** |
| **クリエイティブ** | CEO実名ツイート (取締役対立型 / 数字公開型 / 失敗開示型) をプロモート |
| **オーディエンス** | (1) フォロー類似: 経営者YouTuber/VCパートナー陣のフォロワー / (2) キーワード: 「DX」「経営判断」「中堅企業」「AI活用」 |

**特殊配信**: 重要投稿日 (取締役会後・大型ニュース時) に集中投下 (Burst型)

### 2-3. LINE Ads Platform (LAP) — 補助

**理由**: 中堅企業オーナーの50%以上がLINE 個人利用。リーチ自体は広いが、経営文脈に弱い。

| 項目 | 設定 |
|------|------|
| **目的** | 認知 + リスト構築 (LINE公式アカウント追加) |
| **配置** | LINE News Feed / Tab / Smart Channel |
| **入札** | CPF (Cost Per Friend追加) で安価に友達追加 |
| **予算配分** | 全広告予算の **20%** |
| **クリエイティブ** | バナー (ベネフィット訴求) + 短尺動画 |
| **オーディエンス** | (1) 年齢40-55歳男性・年収1,000万以上 / (2) 興味関心:経営・ビジネス |

**設計**:
- LINE公式アカウント追加 → ステップ配信6通でナーチャリング → 7通目で個別相談予約
- 設楽 ([[ceo-secretary-shidara]]) と紹介プログラム連動

## 3. クリエイティブ戦略 ([[branding-jobs]] [[design-ive]] と協働)

### 3-1. クリエイティブ原則 (Jobs厳守)
- 第一フレーム3秒で「これは普通の広告じゃない」と思わせる
- CEO顔出しは Meta/X はOK、LINEは静止画+テキストでテスト
- 価格を最初に出すか、最後まで隠すかは A/Bテスト
- 「経営者の方へ」「中堅企業の社長へ」のような直接呼びかけは禁止 (NGリスト)
- 派手な装飾 (光・派手色) なし。白基調・モノクロが原則 (Ive 設計)

### 3-2. メッセージ階層 (CEOSNSブランディングと整合)
- **Hook (3秒)**: 「20人の経営チームを、月50万で。」
- **Problem (5秒)**: 「経営判断、全部ひとりで抱えていませんか?」
- **Reveal (10秒)**: 「これが、私が運営する会社です。」(取締役会の動画 or 部長別ダッシュボード)
- **CTA (2秒)**: 「30分の無料相談を予約 / 資料を受け取る」

### 3-3. クリエイティブ本数 (Month 1)
- 動画: 4本 (15秒/30秒/60秒/120秒)
- 静止画バナー: 6本 (Meta 4種 + LINE 2種)
- LP用ヘッダー画像: 3案 (A/Bテスト用)

## 4. LP (Landing Page) 戦略 ([[design-ive]] [[dev-torvalds]])

### 4-1. LP 構造
1. **ファーストビュー**: Tagline + CEOの顔写真 + 30秒動画 + CTA
2. **問題提起**: 中堅企業オーナーの一人経営の限界
3. **解決策**: 20エージェント = 経営チーム
4. **証拠**: 取締役会の様子 (動画 or スクショ)
5. **料金**: Light / Standard / Premium の3プラン
6. **FAQ**: 反論処理TOP20の中から5項目
7. **CTA**: 30分無料相談予約フォーム (Calendly連動)

### 4-2. LP A/B テスト変数 (Month 1)
- ファーストビューの動画 vs 静止画
- CTA文言: 「無料相談」vs「資料請求」vs「デモを見る」
- 料金表示: 3プラン同列 vs Standardを真ん中で強調
- 取締役会の見せ方: 動画 vs スクショ vs 議事録抜粋

### 4-3. LP計測 (dev-torvalds と連携)
- Vercel Analytics + PostHog でユーザー行動可視化
- Heatmap (Hotjar) で離脱箇所特定
- CV後のフォーム入力途中離脱の追跡 (Form Abandonment)

## 5. 計測・最適化プロトコル

### 5-1. 計測タグ実装 (期限 2026-05-30, [[dev-torvalds]])
- Meta Pixel (LP + サンクスページ)
- X Pixel
- LINE Tag
- GA4 (全LP共通)
- Server-side tracking (Conversions API for Meta/X)

### 5-2. 週次最適化 (Kotler 主催, 毎週水曜)
- **金曜まで**: 当週KPIサマリ作成
- **月曜午前**: 翌週クリエイティブ・予算配分決定
- **水曜10:00**: Kotler主催 30分 review with Belfort/Jobs/Ive

### 5-3. 主要KPI

| 指標 | 計算式 | 月次目標 |
|------|--------|----------|
| CPM | 広告費 ÷ インプレッション × 1000 | < 2,000円 (Meta), < 3,000円 (X) |
| CTR | クリック ÷ インプレッション | > 1.5% (Meta), > 1.0% (X) |
| CPL | 広告費 ÷ MQL数 | < 5,000円 (Meta), < 8,000円 (X) |
| CVR (LP→MQL) | MQL ÷ LP訪問 | > 3.0% |
| CVR (MQL→契約) | 契約 ÷ MQL | > 25% (Belfort管理) |
| CAC | 広告費 ÷ 契約数 | < 30万円 |
| LTV/CAC | LTV ÷ CAC | > 3.0 |

### 5-4. 停止トリガー (Buffett決裁ライン)
- 月次 CAC が30万円超過 → 翌週から該当広告セット停止
- 月次 LTV/CAC が3未満で2ヶ月連続 → 該当チャネル全停止
- ブランド毀損コメント (Jobs判断) → 即停止

## 8. 競合参考 (Jobs/Kotler 共同分析)

### 8-1. ベンチマーク広告
- 海外: Claude/Anthropic公式広告 (LinkedIn) — UI簡素・コピー秀逸
- 海外: 経営者向けSaaS (Gong.io, Outreach.io) — トーンの参考
- 国内: 経営コンサルSaaS (Bizmedia, Manegy) — 価格訴求の参考
- 国内: AI 系SaaS (ChatGPT Enterprise 公式, Algomatic) — 訴求差別化の参考

### 8-2. 差別化軸
- 競合は「機能」を訴求 → 当社は「経営チームを丸ごと借りる」**カテゴリそのもの**を訴求
- 競合は「効率化」を訴求 → 当社は「判断の質向上」を訴求
- 競合はモデル名 (GPT-4 / Claude 3) を出す → 当社は出さない (ブランド固有性のため)

## 9. リスクと対処

| リスク | 影響 | 対処 | 監視 |
|--------|------|------|------|
| 広告費爆増 | 大 | 日次予算上限 + 自動停止ルール | Kotler 日次 |
| 不適切ターゲ→ICP外MQL | 中 | Belfort側でフィルタ、Kotlerに精度フィードバック | Kotler 週次 |
| ブランド毀損コピー誤掲載 | 大 | 全クリエ Jobs事前承認 | Jobs 都度 |
| 計測タグ未設置・CV計上漏れ | 大 | dev-torvalds 月次タグ監査 | dev 月次 |
| Meta/X/LINE ポリシー違反停止 | 中 | 「経営助言」直接表現を避ける、法務 ([[legal-kujou]] [[legal-komikado]]) 文言レビュー | 法務 都度 |
| Click Fraud (不正クリック) | 小 | Meta/X 自動検出 + 月次レビュー | Kotler 月次 |
