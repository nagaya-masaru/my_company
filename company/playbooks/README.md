# Playbooks

各事業部が時間をかけて蓄積する「再現可能なやり方」をここに置く。

## 配置 (事業部別dir)

- `keiei/` — 経営全般 (例: `2026-business-plan.md`)
- `sales/` — 営業部
- `marketing/` — マーケティング部
- `branding/` — ブランディング・広報
- `ma/` — M&A・PE

該当が無い場合は新規 dir を作って良い (例: `legal/`, `healthcare/`, `finance/`)。

## 命名規則

`{事業部dir}/{topic}.md`
例: `sales/cold-call-script.md`, `branding/ceo-sns-branding.md`

## 責任

- 各事業部長 (sub-agent) が、自分の領域のプレイブックを書き溜める
- ナレッジ部 [[knowledge-dalio]] が品質と更新をモニタリング
- CEO は新人 (新エージェント or 人間) を入れる時、ここを読ませる
