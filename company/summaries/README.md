# セッションサマリ

長屋CEOと事業部長エージェントの会話セッションの記録。

## ルール

- **配置**: `{事業部dir}/` 配下に置く。dir は内容に最も近い事業部:
  - `keiei/` — 経営全般・組織設計・ハーネス方針
  - `sales/` — 営業・事業計画
  - `marketing/` — マーケティング・広告
  - `branding/` — ブランディング・広報
  - `ma/` — M&A・PE・新規事業
  - `product/` — 商品・オファリング設計
  - 該当が無い場合は新規 dir を作って良い (例: `legal/`, `healthcare/`)
- **命名**: `yyyymmdd_{topic-slug}.md` (旧 `yyyymmdd_summary_{slug}.md` から `summary_` を削除)
- **保存タイミング**: セッション終了直前、メインClaudeが必ず書く
- **対象**: 実質的なやり取りがあった全セッション
- **同日複数セッション**: トピックが異なれば別ファイル。同じならファイル末尾に追記
- **雑談のみで保存不要**: `_trivial/${yyyymmdd}.md` を空で作成 (Stop フックが許可)

## ファイル構造 (`_template.md` 参照)

1. 議題
2. 主な結論
3. 参加した事業部長エージェント
4. アクションアイテム
5. Decision Log項目

詳細は [CLAUDE.md](../../CLAUDE.md) の「セッションサマリ保存ルール」セクション。
