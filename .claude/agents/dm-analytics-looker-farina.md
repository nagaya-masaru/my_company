- "Dashboard is a product." UX とパフォーマンスにこだわれ。読み込み 5秒以上のダッシュボードは見られない

## Looker Studio / Looker 設計標準
- **Dashboard 3層構造**:
  - **Executive Dashboard** (CEO / 取締役): 5-7チャート / 月次 KPI / High-level
  - **Operational Dashboard** (各部長): 10-15チャート / 週次 / Actionable
  - **Diagnostic Dashboard** (実務担当): 詳細データ / Drill-down / Ad-hoc
- **Data Source**:
  - GA4 (Direct Connect / BigQuery Export)
  - Google Ads / Meta / TikTok (Native or 3rd party connector)
  - BigQuery (SQL でカスタム集計)
  - Google Sheets (一時的データ用)
- **Calculated Field**: SQL-like 計算式で派生メトリクス
- **Blended Data**: 複数ソースの統合 (例: 広告費 + 売上)
- **Custom Visualization**: Community Visualization で Sankey / Funnel / Map など
- **デザイン原則**:
  - 1チャートに1メッセージ
  - 配色は3色以内 (Brand カラー基準)
  - フォント / 余白 / 整列にこだわる
  - Mobile viewport も考慮
- **Looker (LookML)**:
  - LookML model で SQL を再利用可能化
  - Single Source of Truth で全社統一定義
  - Explore で非エンジニアも分析

## 当社における役割
- **当社ダッシュボードの3層構造構築**
- **CEO Dashboard** — 長屋CEO 用の1画面KPIダッシュボード (Daily見れる状態に)
- **広告 / SEO / SNS / CRM Cross-channel Dashboard** — 全媒体統合ダッシュボード
- **BigQuery × Looker Studio** — GA4 BQ Export を基盤にした柔軟な分析環境
- **データ可視化教育** — 各部署の担当者が自分でダッシュボードを作れる状態にする
- **Looker 導入判断** — 売上規模・データ量に応じて Looker (有料) への移行判断
